# Frameworks for High-Converting Landing Pages

Use these frameworks as *thinking tools* to structure copy fast, reduce fluff, and increase clarity.

---

## PAS (Problem → Agitate → Solve)

### When to use
- Cold traffic
- Pain-aware audiences
- “This sucks, I need a fix” markets

### Structure
1. **Problem**: Name the pain in plain language.
2. **Agitate**: Make the cost real (time, money, stress, missed outcomes).
3. **Solve**: Present your offer as the clean path out.

### Copy prompts
- Problem: “If you’re doing ___, you’re probably seeing ___.”
- Agitate: “Which means you’re losing ___ every ___.”
- Solve: “___ fixes this by ___, so you can ___.”

### Mini template
- Headline: “Stop ___ Without ___”
- Subhead: “___ helps you ___ in ___ days/weeks — without ___.”
- CTA: “Get ___ / See it in action / Start ___”

---

## AIDA (Attention → Interest → Desire → Action)

### When to use
- Founder-led product pages
- Mid-funnel traffic
- Offer is easy to understand

### Structure
1. **Attention**: Bold outcome or contrarian claim.
2. **Interest**: Explain the “why/how” in 2–3 lines.
3. **Desire**: Benefits + proof + objections answered.
4. **Action**: CTA + risk reversal.

### Copy prompts
- Attention: “Get ___ without ___.”
- Interest: “Here’s how it works: ___.”
- Desire: “So you can ___ + proof (___).”
- Action: “Start ___ / Book ___ / Get ___”

---

## BAB (Before → After → Bridge)

### When to use
- Aspirational outcomes
- “Transformation” offers
- Clear “from/to” story (especially services & subscriptions)

### Structure
1. **Before**: Current reality (frustration, chaos).
2. **After**: Desired future (simple, calm, profitable).
3. **Bridge**: The mechanism (your method/product).

### Copy prompts
- Before: “Right now, you’re stuck with ___.”
- After: “Imagine if ___.”
- Bridge: “With ___, you ___ by ___.”

### Mini template
- Headline: “From ___ to ___”
- Subhead: “A simple ___ that turns ___ into ___.”

---

## 4Ps (Promise → Picture → Proof → Push)

### When to use
- Offer pages with a strong claim
- Paid ads traffic
- When you need a direct, structured hero

### Structure
1. **Promise**: Outcome statement.
2. **Picture**: What life looks like after.
3. **Proof**: Numbers, testimonials, demo, credibility.
4. **Push**: Clear CTA + urgency (only if true).

### Copy prompts
- Promise: “___ in ___ (time) without ___.”
- Picture: “So you can ___.”
- Proof: “Trusted by ___ / Results like ___.”
- Push: “Start ___ today.”

## Objection Handling (The 7 Common Objections)

### How to use
Use these as a checklist for FAQ, proof blocks, and microcopy under CTAs.

1. **Price**: “It’s too expensive.”
   - Reframe cost vs. cost of inaction, ROI, tiering, guarantee.
2. **Time**: “I don’t have time.”
   - Emphasize speed, simplicity, done-for-you, fast setup.
3. **Trust**: “Will this work for me?”
   - Proof, case studies, specifics, demo, transparent limitations.
4. **Complexity**: “This seems hard.”
   - “3 steps” flow, screenshots, concierge onboarding.
5. **Fit**: “This isn’t for my niche.”
   - ICP clarity + “works best for…” + examples.
6. **Switching**: “I already have something.”
   - Migration help, comparison table, “keep your current tool.”
7. **Risk**: “What if it doesn’t work?”
   - Guarantee, cancel anytime, trial, pay-on-results (if applicable).

### Objection copy formulas
- **Feel–Felt–Found**: “You might feel ___. Others felt ___. They found ___.”
- **Acknowledge + Clarify + Reduce risk**: “Totally fair. Here’s how ___. If not, ___.”
- **Specificity beats reassurance**: Replace “easy” with “takes 7 minutes.”

## Angle Library (pick 1 primary per page)

1. **Speed**: “in 48 hours”
2. **Simplicity**: “one request at a time”
3. **Certainty**: “predictable output”
4. **Control**: “approval before anything ships”
5. **Status**: “looks like a bigger brand”
6. **Cost swap**: “cheaper than hiring”
7. **Risk reversal**: “cancel anytime / guarantee”
8. **Unique mechanism**: “our ___ method”

---

## Messaging Rules (non-negotiable)

- One page = one primary promise.
- Lead with outcomes, support with features.
- Specific beats clever.
- Proof should appear before the first major ask.
- CTA should match intent (Book / Start / Get / See).
