# Landing Page Checklists (CRO + Mobile)

Use these before shipping updates.

---

## CRO Checklist (Conversion Rate Optimization)

### Offer Clarity
- [ ] Can someone explain what you sell in 5 seconds?
- [ ] Headline states a clear outcome (not a vague category)
- [ ] Subhead says who it’s for + why it works
- [ ] CTA is obvious and repeated every ~1.5–2 scrolls
- [ ] Page has ONE primary CTA goal (not 3)

### Message-Market Fit Signals
- [ ] ICP is explicitly named (industry/role/company size)
- [ ] Pain is described in the customer’s words
- [ ] Benefits are outcome-led (time saved, revenue, risk reduced)
- [ ] Features support benefits (not the other way around)

### Proof & Credibility
- [ ] Proof appears above the first major “ask”
- [ ] Testimonials mention outcomes (not just “great”)
- [ ] Claims are specific and believable
- [ ] Case study or example output is visible
- [ ] “How it works” is concrete (steps + timelines)

### Objection Handling
- [ ] Pricing objection answered (ROI framing, comparison, value)
- [ ] Time objection answered (setup time, turnaround)
- [ ] Trust objection answered (proof, process, transparency)
- [ ] Risk objection answered (cancel anytime, guarantee if true)
- [ ] Fit objection answered (“works best for…” + examples)

### Friction Reduction
- [ ] Forms ask only for what’s needed
- [ ] CTA microcopy reassures (no spam, cancel anytime, approval step)
- [ ] Avoids jargon / long paragraphs
- [ ] Scannable: bullets, headers, short lines

### Design & Hierarchy
- [ ] One clear visual path: headline → proof → CTA
- [ ] Primary CTA button stands out with contrast
- [ ] Section spacing is consistent
- [ ] No “wall of text” sections
- [ ] Key points are above the fold on common screens

### Pricing & Packaging
- [ ] Pricing is easy to find and understand
- [ ] If multiple tiers: difference is obvious
- [ ] “Who it’s for” included per tier
- [ ] Guardrails are clear (what’s in / out)
- [ ] Cancellation/refund policy summarized near CTA

### Analytics Readiness
- [ ] Track primary conversion event
- [ ] Track CTA clicks (hero, mid, final)
- [ ] Track scroll depth
- [ ] Track form starts vs submits
- [ ] UTM parameters preserved

---

## Mobile Checklist (the “Thumb Test”)

### Above the Fold (mobile)
- [ ] Headline fits in 2–4 lines
- [ ] Subhead is 1–3 short lines
- [ ] Primary CTA visible without scrolling (or within 1 swipe)
- [ ] No giant nav; sticky CTA optional

### Readability
- [ ] Body font ≥ 16px
- [ ] Line height comfortable (1.4–1.7)
- [ ] Paragraphs max 2–3 lines each
- [ ] Bullets used for lists

### Tap Targets
- [ ] Buttons ≥ 44px height
- [ ] Links spaced to avoid mis-taps
- [ ] Forms easy to complete (keyboard types correct)

### Performance
- [ ] Hero image optimized (no 4MB PNGs)
- [ ] Lazy-load below-the-fold images
- [ ] Avoid heavy animations
- [ ] Page feels fast on LTE

### Visual Hierarchy
- [ ] Cards stack cleanly
- [ ] No tiny logos/testimonials that become unreadable
- [ ] Proof blocks remain legible and punchy

### Sticky Elements
- [ ] Sticky CTA doesn’t cover content
- [ ] Popups don’t block reading instantly
- [ ] Exit-intent popups don’t trigger on mobile scroll weirdness

---

## Final Pre-Launch “10 Second” Test
Ask someone cold to look at the hero for 10 seconds, then answer:
1) What is this?
2) Who is it for?
3) What do I get?
4) What do I do next?

If they can’t answer, rewrite the hero.
